Formularios en HTML

Los formularios son la principal manera de recibir datos del usuario. Esta página está estructurada en 3 partes: los tipos de campos, los atributos más importantes y un ejemplo completo.

1. Tipos de Campos (Inputs y más)

Estos son los bloques de construcción de cualquier formulario.

Input de Texto (type="text"): Campo de texto de una línea.

Input de Contraseña (type="password"): Oculta los caracteres escritos.

Input de Email (type="email"): Valida que el texto tenga un formato de email.

Input de Número (type="number"): Campo para valores numéricos.

Input de Fecha (type="date"): Muestra un selector de fechas.

Checkbox (type="checkbox"): Para opciones de sí/no o selección múltiple.

Radio Button (type="radio"): Permite seleccionar una única opción de un grupo. Deben compartir el mismo atributo 'name' para funcionar.

Selector de Archivos (type="file"): Permite al usuario subir uno o más archivos.

Botón de Envío (type="submit"): Envía los datos del formulario.

Área de Texto (<textarea>): Para texto de múltiples líneas.

Lista Desplegable (<select>): Para seleccionar una opción de una lista.

2. Atributos y Etiquetas Esenciales

Los atributos y etiquetas correctos hacen que un formulario sea funcional y accesible.

La etiqueta <label>

Es la etiqueta de texto para un campo. Es crucial para la accesibilidad. Se vincula a un campo usando el atributo for, que debe coincidir con el id del campo.

Atributo name

Identifica el dato cuando se envía al servidor. Es el "nombre de la variable". Sin este atributo, el dato del campo no se envía.

Atributo value

Define el valor por defecto de un campo o el valor que se enviará. En los radio buttons y checkboxes, es el valor que se envía si la opción está seleccionada.

Atributo placeholder

Muestra un texto de ayuda dentro del campo, que desaparece cuando el usuario empieza a escribir.

Atributo required

Hace que un campo sea obligatorio. El navegador no permitirá enviar el formulario si este campo está vacío.

Atributos del <form>: action y method

action: La URL del servidor que recibirá y procesará los datos.

method: El método HTTP para enviar los datos. Comúnmente GET (para datos no sensibles, visibles en la URL) o POST (para datos sensibles, enviados en el cuerpo de la petición).

<form action="/registrar" method="POST">...</form>
